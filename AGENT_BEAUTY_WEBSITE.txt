ГЛАВНОЕ ПРАВИЛО BEAUTY-ДИЗАЙНА
================================================================================

Сайт должен выглядеть как продолжение интерьера салона, а не как шаблон
«за 500 рублей».

СНАЧАЛА — анализ фото интерьера (vnutri/, snaruzhi/, vhod/).
ПОТОМ — палитра и типографика.
ПОТОМ — вёрстка.

Если в выгрузке есть фото залов (кремовые стены, rose gold, мрамор,
зеркала с лампочками, геометрия сот) — сайт ОБЯЗАН повторять эту эстетику:
  • светлый кремовый/ivory фон, НЕ тёмно-серый generic;
  • акценты металлик (rose gold, champagne, copper) — из реального интерьера;
  • много воздуха, glassmorphism, тонкие линии;
  • динамический фон из фото интерьера (crossfade / ken burns), не статичный цвет.

ЗАПРЕЩЕНО (выглядит дёшево):
  ✗ Тёмный hero-overlay поверх светлого салона
  ✗ Ярко-красные кнопки без связи с интерьером
  ✗ Generic шрифты Arial / Roboto без характера
  ✗ Плоская сетка 3 колонки без иерархии
  ✗ Stock-фото и выдуманные пути к несуществующим файлам
  ✗ Фейковые скидки, таймеры, «осталось 2 места»
  ✗ Копирование чужого сайта клиента

================================================================================
ШАГ 2.5. DESIGN DNA — ЗАПРЕТ НА ПОВТОРЫ (обязательно)
================================================================================

Каждый салон получает УНИКАЛЬНЫЙ сайт. Два сайта из этой системы не должны
выглядеть как один шаблон с перекрашенными кнопками.

ПЕРЕД вёрсткой:
  1. Открой sites/_registry.md (реестр всех сделанных сайтов).
     Если файла нет — создай его.
  2. Прочитай Design DNA всех предыдущих сайтов.
  3. Сформулируй Design DNA нового сайта так, чтобы он отличался от КАЖДОГО
     предыдущего минимум по 4 из 7 осей:

  ОСИ РАЗЛИЧИЯ:
  ┌ 1. Композиция hero: fullscreen-слайдшоу / split 50-50 / асимметрия
  │    с выносом текста / видео-стиль ken burns / коллаж из фото
  ├ 2. Пара шрифтов: не повторяй связку (Cormorant+Manrope уже была? —
  │    возьми Prata+Jost, Marcellus+Onest, EB Garamond+Golos Text...)
  ├ 3. Сетка услуг: табы / аккордеон / прайс-колонки / карточки с фото /
  │    горизонтальный скролл категорий
  ├ 4. Форма галереи: masonry / bento / карусель / filmstrip / полароид-стопка
  ├ 5. Декоративный мотив: соты / арки / лампочки / мрамор / ботаника /
  │    линии-волны — берётся ИЗ ИНТЕРЬЕРА конкретного салона
  ├ 6. Ритм секций: чередование светлое/тёмное, цветные разделители,
  │    диагональные срезы, overlap-карточки
  └ 7. Подача отзывов: карусель / чат-пузыри / рукописные карточки /
       большая цитата + мелкие

  4. Design DNA диктуется САЛОНОМ, не твоим вкусом: барбершоп в лофте ≠
     ногтевая студия в пастели ≠ SPA в мраморе. Если два салона реально
     похожи интерьером — различай композицией, шрифтами, мотивом.

  5. После сдачи сайта ДОПИШИ строку в sites/_registry.md:

     | slug | город | салон | настроение | палитра (3 hex) | шрифты | hero | мотив | дата |

ЗАПРЕЩЕНО: копировать структуру index.html предыдущего сайта через
find-replace цветов. Каждый сайт верстается от Design DNA.

Используй минимум 4 из списка (подходящих к конкретному салону):

  □ Mirror-card — рамка превью как зеркало с подсветкой

Типографика:
  — Заголовки: serif, крупный кегль, italic для акцентных слов
  — Текст: light sans, letter-spacing на лейблах (0.2em uppercase)
  — Межстрочный интервал щедрый (1.6–1.75)

ШАГ 6. СТРУКТУРА ФАЙЛОВ САЙТА
================================================================================

Собери в папку sites/<slug>/:

  sites/<slug>/
    index.html
    styles.css
    app.js
    data.js          ← menu, reviews, пути к фото (JSON из выгрузки)
    PITCH.md         ← пакет для продажи (см. ШАГ 7.5)
    OLD_SITE_AUDIT.md ← только в режиме Б (аудит старого сайта)
    assets/
      interior/      ← копии фото vnutri/, vhod/, snaruzhi/ (hero, фон)
      photos/        ← копии uslugi/ или photo_*.jpg (портфолио)

  sites/_registry.md ← общий реестр Design DNA (см. ШАГ 2.5) — дополни!

data.js — единый источник данных для app.js:
  SITE_DATA = { menu: [...], reviews: [...], photos: { base, interior: [...], gallery: [...] } }

Пути в HTML — относительные от sites/<slug>/.
Проверь, что каждый src= указывает на существующий файл.

================================================================================
ШАГ 7. КОД И КАЧЕСТВО
================================================================================

  • Семантический HTML5, meta description, viewport
  • Адаптив: 375px → desktop
  • tel:+7... на всех кнопках записи
  • target="_blank" rel="noopener noreferrer" на внешних ссылках
  • Lightbox через <dialog> или overlay
  • EscapeHtml для данных из JSON (XSS)
  • prefers-reduced-motion — отключить анимации
  • Нет console.error, битых путей, пустых секций

СКОРОСТЬ (клиент откроет сайт с телефона при созвоне — он ОБЯЗАН влетать):
  • Сжимай фото: hero ≤ 250 КБ, галерея ≤ 150 КБ на файл (resize до 1600px
    по длинной стороне; если есть ffmpeg/PIL — конвертируй тяжёлые в .webp)
  • loading="lazy" на всех фото ниже первого экрана
  • width/height у <img> — чтобы страница не прыгала
  • Шрифты: preconnect к fonts.googleapis.com, display=swap, максимум 2 семейства

SEO И ШЕРИНГ (сайт будут пересылать в WhatsApp — превью должно быть красивым):
  • <title>: «<Салон> — салон красоты в <городе> | <2–3 ключевые услуги>»
  • meta description с услугами и адресом (150–160 символов)
  • Open Graph: og:title, og:description, og:image (лучшее фото интерьера)
  • JSON-LD schema.org/BeautySalon: name, address, telephone, openingHours,
    aggregateRating (рейтинг и число отзывов — строго из profile.json)
  • favicon (можно простой — буква названия на цвете Accent)

ШАГ 7.5. ПАКЕТ ДЛЯ ПРОДАЖИ (PITCH.md — обязательный файл)
================================================================================

Сайт продаёт не себя — его продают на созвоне. Вместе с сайтом собери
sites/<slug>/PITCH.md для владельца бизнеса:

  1. Скрипт показа (3–5 реплик): что показать первым (hero с ИХ интерьером),
     на чём акцент (их реальные отзывы, их цены, запись в 1 клик).
  2. Таблица «Карточка на Яндексе / старый сайт → Новый сайт» — 5–7 строк
     конкретных улучшений. В режиме Б — по пунктам OLD_SITE_AUDIT.md.
  3. Ответы на 3 типовых возражения:
     «у нас и так есть клиенты» / «дорого» / «нам некогда этим заниматься».
  4. Что нужно от клиента для запуска: домен, подтвердить цены,
     фото мастеров (по желанию), 30 минут времени.
  5. Апселлы вторым платежом: онлайн-запись (YClients/DIKIDI), домен+хостинг,
     обновление цен раз в месяц, Яндекс.Бизнес оформление.

Тон PITCH.md — для владельца салона, без технических терминов.

Превью локально (если есть parser-kart):
  py scripts/serve_preview.py --open sites/<slug>/
  URL: http://127.0.0.1:8777/sites/<slug>/
  НЕ использовать порт 8765 (конфликт с другими сервисами).
