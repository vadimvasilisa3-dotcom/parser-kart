# ТЗ: Калькулятор потерь салона «7 дыр»

**Версия:** 1.0  
**Назначение:** инструмент продажи коробки 1 000–3 000 ₽ — показать владельцу **₽/мес и ₽/год** операционных потерь без рекламы.  
**Файл v1:** `index.html` (standalone, без бэкенда)

---

## 1. Цель продукта

| | |
|---|---|
| **Для кого** | Владелец / администратор салона 2–15 мастеров, не маркетолог |
| **Задача** | За 2–3 минуты увидеть, **сколько денег утекает** в рутине |
| **Не задача** | Заменить CRM, финансовый аудит, бухгалтерию |
| **KPI успеха** | 80% пользователей понимают итог без объяснения; 50% сохраняют/печатают PDF |

---

## 2. Принципы расчёта (обязательные)

1. **Прозрачность** — под каждой строкой видна формула с подставленными числами.
2. **Консервативность** — дефолты из **нижней** части отраслевого диапазона, где есть спор.
3. **Пересечения** — итог умножается на **0,88** (−12%), т.к. no-show и низкая загрузка частично дублируются.
4. **Два режима выручки:**  
   - авто: `мастера × записи/день × дни × чек`  
   - ручной: поле «выручка в месяц» перебивает авто
5. **Оценки при 0:** база = `визиты_факт × 12 × 0,7`; обращения = `записи_план × 0,25`
6. **Валюта:** только ₽, округление до целых, формат `1 234 567 ₽`

## 4. Формулы потерь (₽/мес)

Обозначения:
- `P` = `masters × appts × days` — плановые записи/мес  
- `R` = выручка/мес (ручная или `P × check`)  
- `F` = `P × (1 − noshow%)` — состоявшиеся визиты  
- `N` = `F × newShare%` — новых клиентов/мес  

### Зона 1. No-show
```
Loss₁ = P × (noshow/100) × check
```

### Зона 2. Низкая загрузка
```
если util < utilTarget:
  Loss₂ = R × (utilTarget/util − 1)
иначе Loss₂ = 0
```

### Зона 3. Спящая база
```
B = base || F × 12 × 0.7
Loss₃ = B × (sleepPct/100) × (reactRate/100) × check
```
*Консервативно: 1 визит на вернувшегося, без LTV.*

### Зона 4. Провал 1→2 визит
```
Loss₄ = N × (drop12/100) × max(0, visitsYear − 1) × check / 12
```
*Упущенные будущие визиты, приведённые к месяцу.*

### Зона 5. Сожжённый CPL
```
если cpl > 0:
  Loss₅ = N × (drop12/100) × cpl
```

### Зона 6. Пропущенные обращения
```
L = leads || P × 0.25
Loss₆ = L × (missedPct/100) × (leadConv/100) × check
```

### Зона 7. Недореализованный ритейл (упущенная прибыль)
```
если retailTarget > retailPct:
  S = R × (1 − retailPct)
  Δоборот = S × (retailTarget − retailPct) / (1 − retailTarget)
  Loss₇ = Δоборот × (retailMargin/100)
```

### Зона 8 (доп.). Низкая перезапись
```
если prebookNorm > prebook:
  extraNoShow = (prebookNorm − prebook) × 0.25
  Loss₈ = F × extraNoShow × check
```
*Коэффициент 0.25 — эмпирика: без перезаписи выше риск no-show.*

### Итог
```
Total_raw = Σ Lossᵢ
Total = Total_raw × 0.88
Total_year = Total × 12
```

## 5. UI/UX требования

### 5.1. Экраны (v1 — один скролл)
1. Шапка: название + 1 строка «не реклама, операционка»
2. Пресеты
3. 4 карточки ввода (2×2 на desktop)
4. **Красный блок итога:** ₽/мес крупно, ₽/год ниже, пояснение про −12%
5. Таблица 7–8 строк: название | сумма | bar | формула | источник
6. Сноски с URL-источниками

### 5.2. Поведение
- Пересчёт **on input** без кнопки «Посчитать»
- Сортировка строк по убыванию суммы
- `@media print` — скрыть пресеты, сохранить итог и формулы
- Mobile-first, min-width полей 100%

### 5.3. Запрещено
- Обещания «вы сэкономите X» без формулы
- Скрытые коэффициенты
- Обязательная регистрация в v1

## 9. Sanity-check (дефолт: 5 мастеров, 8 записей, 26 дней, чек 2500)

| Показатель | Значение |
|------------|----------|
| Плановых записей | 1 040 |
| Выручка (план) | 2 600 000 ₽ |
| Loss₁ no-show 20% | 520 000 ₽ |
| Loss₂ util 65→80% | 600 000 ₽ |
| Loss₃ спящие (база 1200) | 135 000 ₽ |
| Loss₄ 1→2 визит | 148 512 ₽ |
| Loss₅ CPL | 81 469 ₽ |
| Loss₆ пропущенные | 60 500 ₽ |
| Loss₇ ритейл | 179 400 ₽ |
| Loss₈ перезапись | 104 000 ₽ |
| **Total_raw** | **1 828 881 ₽** |
| **Total × 0.88** | **1 609 416 ₽/мес** |
| **Total × 12** | **19,3 млн ₽/год** |

*Цифра шокирует — это фича для продажи. На созвоне показываем **топ-3 строки**, не весь итог.*

## 10. Скрипт показа (5 реплик)

1. «Сколько мастеров и какой средний чек?» → вбиваем вместе  
2. «No-show примерно сколько из 10?» → 2 = 20%  
3. «Вторник днём пусто или всё забито?» → загрузка  
4. Показываем **3 самые жирные строки** с формулами  
5. «Вот здесь деньги без рекламы. Реклама без этого — дырявое ведро.»
